# Fix: pnpm workspace "packages field missing or empty" error

## Problem
`pnpm-workspace.yaml` in `/home/z/Public/go-bin/app/` contains only `allowBuilds` but is missing the required `packages` field. pnpm 9.7.1 sees the workspace file, enters workspace mode, and rejects commands that validate workspace config (like `cap run android`). Scripts that don't trigger workspace validation (`pnpm build`) work fine.

## Fix
Add `packages: []` to `/home/z/Public/go-bin/app/pnpm-workspace.yaml`:

```yaml
packages: []
allowBuilds:
  sharp: true
```

This tells pnpm "workspace root with no child packages", satisfying the validation while preserving the `allowBuilds` setting for sharp.

## File to modify
- `/home/z/Public/go-bin/app/pnpm-workspace.yaml` — add `packages: []`

## Verification
```bash
cd /home/z/Public/go-bin/app
pnpm run android:run
```
